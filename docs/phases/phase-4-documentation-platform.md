# Phase 4 — Documentation Platform

**Status:** ✅ Complete

## Goal

Build the public ETHDS documentation site so design principles, brand
system, components, patterns, and contribution guidance are discoverable
and usable by government teams and contributors.

## Deliverables

All delivered in [`packages/ethds-docs/`](../../packages/ethds-docs/):

- Docusaurus Setup — `@docusaurus/preset-classic`, TypeScript config
- Site Architecture — landing page + docs (Introduction, Design
  Principles, Brand System, Accessibility, Contributing)
- Navigation — autogenerated sidebar, navbar, footer
- Search — offline/local full-text search
  (`@easyops-cn/docusaurus-search-local`), no external service
- Multilingual Documentation — i18n configured for all six ETHDS locales
  (`en, am, om, ti, so, aa`); English authored, others selectable and
  falling back to English until translated (per
  [Phase 6](phase-6-localization-framework.md))
- Accessibility Documentation — published accessibility statement; the
  site is themed from `@ethds/tokens` and targets WCAG 2.2 AA
- Deployment Pipeline + GitHub Pages —
  [`.github/workflows/docs-deploy.yml`](../../.github/workflows/docs-deploy.yml)

The site theme is driven by `@ethds/tokens` ([Phase 3](phase-3-design-tokens.md)),
so the documentation visibly uses the design system it documents.

## Output Location

[`packages/ethds-docs/`](../../packages/ethds-docs/) (replaced the former
README-only stub).

## Dependencies

- [Phase 1 — Design Principles](phase-1-design-principles.md) through
  [Phase 3 — Design Tokens](phase-3-design-tokens.md) for initial content
  to publish. Content from later phases (5–9) is added incrementally as
  those phases complete.
