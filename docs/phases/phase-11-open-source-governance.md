# Phase 11 — Open Source Governance

**Status:** 📋 Planned

## Goal

Formalize the lightweight governance baseline established in
[Phase 0](phase-0-foundation.md) (see
[`docs/governance/governance-model.md`](../governance/governance-model.md))
into the full process needed to sustain ETHDS as it grows beyond a small
founding team.

## Deliverables

- RFC Process (formalizing the process referenced from
  [`CONTRIBUTING.md`](../../CONTRIBUTING.md#proposing-significant-changes)
  and archived in `packages/ethds-rfcs/`)
- Component Review Process
- Accessibility Review Process (operationalizing
  [Phase 5](phase-5-accessibility-framework.md))
- Localization Review Process (operationalizing
  [Phase 6](phase-6-localization-framework.md))
- Security Policy
- Release Management (building on
  [`docs/governance/release-strategy.md`](../governance/release-strategy.md))
- Maintainer Responsibilities
- Contributor Ladder (formalizing
  [`docs/governance/community-model.md`](../governance/community-model.md))
- Community Guidelines
- Long-Term Sustainability Plan

## Output Location

`docs/governance/` (expanding the Phase 0 baseline files) and
`packages/ethds-rfcs/`.

## Dependencies

- [Phase 0 — Foundation](phase-0-foundation.md) for the governance and
  community baseline this phase formalizes.
- Benefits from having real usage history from
  [Phase 7](phase-7-core-components.md) through
  [Phase 10](phase-10-reference-implementations.md) to govern around —
  process designed before any real components/patterns/contributors
  exist tends to need revision anyway.
