# Governance

How ETHDS makes decisions, reviews changes, grows contributors into
maintainers, and ships releases — all in the open. ETHDS uses a
lightweight, RFC-driven model: small changes go through normal pull
request review, while significant changes (new components, patterns,
templates, breaking changes, or governance itself) go through a public
[RFC](rfc-process.md) with required accessibility and localization
sign-off where relevant.
