Check the state of a submitted request. States are conveyed in plain
language, never an internal workflow code, and never by colour alone.

```tsx
import { StatusView } from '@ethds/patterns';

<StatusView
  reference="ETH-2026-0041"
  statusText="Approved — ready for pickup"
  detail="Your passport is ready for pickup at the Bole office."
  variant="success"
  timeline={[
    { label: 'Submitted', date: 'Meskerem 3, 2019 EC' },
    { label: 'Approved', date: 'Meskerem 9, 2019 EC' },
  ]}
/>
```

## When to use this pattern

Use Status Tracking wherever a citizen needs to check on a submitted
application, complaint, or booking — typically linked from the
Citizen Dashboard template's applications table.

## Validation & error handling

There's no form here — `StatusView` is a display-only component. The
caller is responsible for fetching the current `statusText`, `detail`,
and `timeline` and mapping any internal workflow state to plain,
citizen-facing language before rendering.

## Accessibility

- The status is conveyed via the `Alert`'s `variant` **and** a labelled
  icon **and** text — never colour alone.
- `iconLabel` defaults to `statusText` so the icon's accessible name always
  matches what's displayed; override it if you need a distinct
  announcement.
- The timeline renders as a semantic ordered list (`<ol>`), so assistive
  technology can announce each step's position and count.

## Plain HTML

A display-only page — reuses [Alert](/docs/components/alert#plain-html)'s
markup for the current status plus a plain `<ol>` for the timeline. See
the [pattern's own HTML Example](https://github.com/yidnekachewkibru/ethgov-design-system/blob/main/packages/ethds-patterns/patterns/status-tracking.md#html-example)
for the full markup.

## Localization

`statusText`, `detail`, and each timeline step's `label`/`date` are all
passed in as translated content — dates should already be formatted per
the [Ethiopian calendar](https://github.com/yidnekachewkibru/ethgov-design-system/blob/main/docs/localization/date-formatting.md)
convention before reaching this component.

## Props

| Prop | Type | Required |
|---|---|---|
| `reference` | `string` | Yes |
| `statusText` | `string` | Yes |
| `detail` | `ReactNode` | Yes |
| `timeline` | `StatusStep[]` | Yes |
| `variant` | `'success' \| 'warning' \| 'error' \| 'info'` (default `'success'`) | No |
| `iconLabel` | `string` (default: `statusText`) | No |
| `labels` | `Partial<StatusViewLabels>` | No |
