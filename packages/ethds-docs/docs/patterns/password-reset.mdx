Recover access to an account in three steps — request, verify, set a new
password — built on the shared `useMultiStepForm` hook. Try the full flow
above: every step in this demo succeeds.

```tsx
import { PasswordResetFlow } from '@ethds/patterns';

<PasswordResetFlow
  onRequestReset={async (identifier) => {
    /* send the code; never reject just because the identifier
       doesn't match an account — that would leak whether it exists */
  }}
  onVerifyCode={async (code) => {
    /* verify with your backend */
  }}
  onSubmit={async (data) => {
    /* set the new password; data: PasswordResetData */
    return { reference: 'ok' };
  }}
/>
```

## When to use this pattern

Use Password Reset when a citizen can't sign in and needs to regain
access. Step 2 reuses the [OTP Verification](otp-verification) pattern
directly, so the two stay in sync automatically.

## Validation & error handling

- **Step 1 (request):** never branches on whether the identifier matches
  an account — the non-revealing wording lives entirely in step 2's "we
  sent a code to…" message, so nothing in the flow can leak account
  existence.
- **Step 3 (set new password):** minimum 8 characters; paste and password
  managers are allowed (`autocomplete="new-password"`).
- A failed final submit shows an error with a **Try again** button rather
  than losing the citizen's progress or bouncing them back to step 1.

## Accessibility

- Focus moves to the top of each new step on every transition.
- Other signed-in devices are said to be signed out on completion — stated
  in plain language on the confirmation panel, a security-relevant fact
  the citizen needs to know.

## Plain HTML

Like [Application Submission](application-submission#plain-html),
`PasswordResetFlow`'s three steps become three pages without a client
framework — `/reset`, `/reset/verify` (exactly
[OTP Verification's markup](otp-verification#plain-html)), and
`/reset/new-password` — with the identifier and verified state held in
the server session. See the
[pattern's own HTML Example](https://github.com/yidnekachewkibru/ethgov-design-system/blob/main/packages/ethds-patterns/patterns/password-reset.md#html-example)
for the full three-page write-up.

## Localization

Every step's labels are translatable via the flow's `labels` prop, which
nests each step's own label type (`requestReset`, `otp`,
`setNewPassword`, `confirmation`).

## Props

| Prop | Type | Required |
|---|---|---|
| `onRequestReset` | `(identifier: string) => Promise<void>` | Yes |
| `onVerifyCode` | `(code: string) => Promise<void>` | Yes |
| `onSubmit` | `(data: PasswordResetData) => Promise<SubmitResult>` | Yes |
| `onSaveDraft` | `(data: Partial<PasswordResetData>) => void \| Promise<void>` | No |
| `labels` | `Partial<PasswordResetFlowLabels>` | No |
