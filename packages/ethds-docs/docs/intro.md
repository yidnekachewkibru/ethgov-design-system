# Ethiopian Government Design System (ETHDS)

ETHDS is an open-source design system for Ethiopian government digital
services — inspired by the GOV.UK Design System, the U.S. Web Design
System, and the Cyprus Government Unified Design System.

It provides the shared design principles, brand system, design tokens,
components, patterns, and templates that let ministries, agencies,
regional and municipal governments, and public institutions build
consistent, accessible, multilingual services — without each team solving
the same problems from scratch.

## What ETHDS gives you

- **[Design principles](/docs/principles)** — the ten rules every ETHDS
  decision is measured against.
- **[Brand system](/docs/brand)** — colour, typography, spacing, and more,
  with verified accessible contrast.
- **Design tokens** — the `@ethds/tokens` package: one source of truth,
  built to CSS variables, TypeScript, and a Tailwind preset.
- **Components, patterns, templates** — arriving in later phases.

## Principles in brief

1. Start with citizen needs
2. Multilingual by default
3. Mobile first
4. Accessibility by default
5. Simple before powerful
6. Design for low bandwidth
7. Reuse before building
8. Security and privacy by design
9. Consistent government experience
10. Open by default

## A note on source of truth

This site is the public face of ETHDS. The canonical, full specifications
live in the project repository under `docs/` and in each package. Where a
page here summarises, it links to the canonical source on GitHub so there
is a single authoritative version.

## Languages

ETHDS targets six languages — English, Amharic, Afaan Oromo, Tigrinya,
Somali, and Afar. This site is authored in English; the other languages
are selectable from the language menu and fall back to English until
their translations are contributed.
