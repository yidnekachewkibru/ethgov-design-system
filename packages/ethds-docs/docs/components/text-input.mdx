# Text input

A labelled single-line field for short answers — names, phone numbers,
reference codes.

<Example>
  <TextInput label="Given name" autoComplete="given-name" />
</Example>

```tsx
import { TextInput } from '@ethds/react';

<TextInput label="Given name" autoComplete="given-name" />
```

## When to use this component

Use for a single line of text. Label with a clear question or field name
in the citizen's language. For Ethiopian names, use "Given name" /
"Father's name" — never "Surname" (see the address & name standards).

## When not to use this component

Don't use for multi-line answers ([Textarea](textarea)), for a fixed set
of options ([Radios](radios) / [Select](select)), or without a visible
label.

## Hint and error

<Example>
  <div style={{ display: 'grid', gap: '1rem' }}>
    <TextInput
      label="Phone number"
      type="tel"
      inputMode="tel"
      autoComplete="tel"
      hint="Format: 09xx xxx xxx"
      required
    />
    <TextInput
      label="Email"
      defaultValue="not-an-email"
      error="Enter a valid email address"
    />
  </div>
</Example>

The hint explains the format **before** the citizen types; the error says
what's wrong and how to fix it. Both are announced by screen readers.

## Accessibility

- The visible `label` is programmatically associated (`htmlFor`) —
  required, not optional.
- Hint and error are linked via `aria-describedby`; an error also sets
  `aria-invalid="true"`.
- 44px minimum height; the functional border uses `gray-500` (≥3:1
  non-text contrast).
- Use correct `autocomplete`/`inputMode` (`tel`, `email`, `numeric`) so
  autofill and mobile keyboards work.

## Plain HTML

The label/hint/error wiring below is the pattern every ETHDS form field
follows — the same `for`/`id` and `aria-describedby` convention appears
on Textarea, Select, Checkbox, Radio, and every other field, so it's
described in full once here rather than repeated on each of their pages.

```html
<div class="ethds-field">
  <label for="phone" class="ethds-label">Phone number</label>
  <span id="phone-hint" class="ethds-hint">Format: 09xx xxx xxx</span>
  <input
    id="phone"
    name="phone"
    type="tel"
    inputmode="tel"
    autocomplete="tel"
    class="ethds-input"
    aria-describedby="phone-hint"
    required
  />
</div>

<!-- With an error: aria-describedby lists BOTH ids (space-separated)
     when a hint and an error are both present, and aria-invalid is set. -->
<div class="ethds-field">
  <label for="email" class="ethds-label">Email</label>
  <input
    id="email"
    name="email"
    type="email"
    class="ethds-input ethds-input--error"
    aria-describedby="email-error"
    aria-invalid="true"
  />
  <span id="email-error" class="ethds-error">Enter a valid email address</span>
</div>
```

```css
.ethds-field { display: flex; flex-direction: column; gap: var(--ethds-space-1); }
.ethds-label { font: var(--ethds-font-medium) var(--ethds-text-base) var(--ethds-font-sans); }
.ethds-hint { font-size: var(--ethds-text-sm); color: var(--ethds-color-text-secondary); }
.ethds-input {
  min-height: 44px;
  padding: var(--ethds-space-2) var(--ethds-space-3);
  border: 1px solid var(--ethds-color-border);
  border-radius: var(--ethds-radius-md);
}
.ethds-input:focus-visible {
  outline: 2px solid var(--ethds-color-focus);
  border-color: var(--ethds-color-focus);
}
.ethds-input--error { border-color: var(--ethds-color-error); }
.ethds-error {
  font-size: var(--ethds-text-sm);
  font-weight: var(--ethds-font-medium);
  color: var(--ethds-red-700);
}
```

No JavaScript is needed for the field itself. Server-side, re-render the
same markup with the error span and `aria-invalid` present when
validation fails — the field never loses the citizen's entered value.

## Props

| Prop | Type | Notes |
|---|---|---|
| `label` | `ReactNode` | required, translatable |
| `hint` | `ReactNode` | optional help text |
| `error` | `ReactNode` | marks the field invalid |
| `required` | `boolean` | adds a visible "(required)" marker |
| …plus all native `<input>` props | | |
