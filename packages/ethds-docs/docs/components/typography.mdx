# Typography

Headings and body text on the ETHDS type scale, with fonts that cover
both Ge'ez script (Amharic, Tigrinya) and Latin script.

<Example>
  <Heading level={1}>Apply for a passport</Heading>
  <Text size="lg">Who this service is for and what it does.</Text>
  <Text>
    Standard body copy at the base size — never smaller than a 16px
    equivalent for primary reading content.
  </Text>
  <Text secondary>Secondary, de-emphasised text.</Text>
</Example>

```tsx
import { Heading, Text } from '@ethds/react';

<Heading level={1}>Apply for a passport</Heading>
<Text>Standard body copy.</Text>
<Text secondary>Secondary text.</Text>
```

## When to use this component

Use `Heading` for every page and section title, choosing `level` for the
**document outline** (one `h1` per page, no skipped levels). Use `Text`
for body copy.

## When not to use this component

Don't pick a heading level for its size — if the visual size must differ
from the outline, keep the semantic `level` and set `visualLevel`:

<Example>
  <Heading level={3} visualLevel={1}>Looks like an h1, still an h3 for screen readers</Heading>
</Example>

## Accessibility

- Real `h1`–`h4` elements — screen-reader users navigate by heading.
- Sizes in `rem`, so text respects the citizen's browser font-size
  preference and 200% zoom.
- The font stack pairs **Noto Sans** with **Noto Sans Ethiopic**, with a
  `system-ui` fallback so text stays legible if web fonts fail on a slow
  connection.

## Plain HTML

Plain semantic elements with an ETHDS class per size — no component
needed.

```html
<h1 class="ethds-h1">Apply for a passport</h1>
<p class="ethds-text ethds-text--lg">Who this service is for and what it does.</p>
<p class="ethds-text">Standard body copy at the base size.</p>
<p class="ethds-text ethds-text--secondary">Secondary, de-emphasised text.</p>

<!-- Looks like an h1, still an h3 in the document outline: -->
<h3 class="ethds-h1">Looks like an h1, still an h3 for screen readers</h3>
```

```css
.ethds-h1, .ethds-h2, .ethds-h3, .ethds-h4 {
  margin: 0; font-family: var(--ethds-font-sans); font-weight: var(--ethds-font-bold);
}
.ethds-h1 { font-size: var(--ethds-text-4xl); line-height: var(--ethds-leading-tight); }
.ethds-h2 { font-size: var(--ethds-text-3xl); line-height: var(--ethds-leading-snug); }
.ethds-h3 { font-size: var(--ethds-text-2xl); line-height: var(--ethds-leading-snug); }
.ethds-h4 { font-size: var(--ethds-text-xl); line-height: var(--ethds-leading-normal); }
.ethds-text { margin: 0; font-family: var(--ethds-font-sans); line-height: var(--ethds-leading-relaxed); }
.ethds-text--lg { font-size: var(--ethds-text-lg); }
.ethds-text--secondary { color: var(--ethds-color-text-secondary); }
```

The class picks the **size**; the tag you choose picks the **document
outline position**. Keep one `<h1>` per page and don't skip levels — pick
the tag for where the heading sits in the outline, and the class for how
big it should look, exactly as the React version's `level` /
`visualLevel` split does.

## Props

| Component | Key props |
|---|---|
| `Heading` | `level: 1–4` (required), `visualLevel?: 1–4` |
| `Text` | `size: 'sm' \| 'base' \| 'lg'`, `secondary?: boolean`, `as: 'p' \| 'span' \| 'div'` |
