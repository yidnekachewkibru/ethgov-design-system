# Table

A semantic data table for structured records — applications, payments,
appointments.

<Example>

## When to use this component

Use for genuinely tabular data — rows of records with the same fields.
Right-align money columns; format amounts as Birr with `Intl`.

## When not to use this component

Don't use tables for page layout, or for a label/value summary of one
record (a description list — and, coming next, a Summary List component —
fits better).

## Accessibility

- A real `<table>` with a `<caption>` (its accessible name) and
  `<th scope="col">` headers — the structure screen-reader users navigate
  by.
- On small screens the table scrolls horizontally inside its own
  container rather than breaking the page (reflow).
- Status text in cells is text, not colour-coded chips alone.
