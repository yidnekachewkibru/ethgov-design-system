# File upload

A labelled file picker, for scanned documents and photos citizens attach
to an application.

<Example>
  <FileUpload label="Upload your Fayda ID scan" hint="JPG, PNG or PDF, up to 5MB" />
</Example>

```tsx
import { FileUpload } from '@ethds/react';

<FileUpload
  label="Upload your Fayda ID scan"
  hint="JPG, PNG or PDF, up to 5MB"
  accept="image/jpeg,image/png,application/pdf"
/>
```

## When to use this component

Whenever a service needs a citizen to attach a document — an ID scan, a
supporting letter, a photo of a permit. State the accepted file types and
any size limit in the hint.

## When not to use this component

Don't require a file when a citizen could type the same information into
a text field — a photo of a form is harder for staff to process than
structured data, and harder for the citizen to produce on a low-end phone
with a poor camera.

## Accessibility

- A native `<input type="file">` — the device's own file/photo picker
  opens, which citizens already know how to use, keyboard- and
  screen-reader-operable for free.
- The chosen file name is echoed in a polite live region, so
  screen-reader users get clear confirmation of what was selected.
- Label, hint, and error are wired via `aria-describedby` / `aria-invalid`,
  the same pattern as [Text input](text-input.mdx).

## Plain HTML

```html
<div class="ethds-field">
  <label for="id-scan" class="ethds-label">Upload your Fayda ID scan</label>
  <span id="id-scan-hint" class="ethds-hint">JPG, PNG or PDF, up to 5MB</span>
  <input
    id="id-scan"
    name="id-scan"
    type="file"
    accept="image/jpeg,image/png,application/pdf"
    class="ethds-file-input"
    aria-describedby="id-scan-hint id-scan-status"
  />
  <span id="id-scan-status" class="ethds-hint" aria-live="polite">No file chosen</span>
</div>
```

```css
.ethds-file-input { padding: var(--ethds-space-2) 0; }
.ethds-file-input:focus-visible { outline: 2px solid var(--ethds-color-focus); outline-offset: 1px; }
```

The browser's own picker already announces the chosen filename in most
screen readers, but a small script gives every citizen the same explicit
confirmation the React version does:

```js
document.querySelectorAll('.ethds-file-input').forEach((input) => {
  const status = document.getElementById(input.getAttribute('aria-describedby').split(' ')[1]);
  input.addEventListener('change', () => {
    status.textContent = input.files.length
      ? `${Array.from(input.files).map((f) => f.name).join(', ')} — file chosen`
      : 'No file chosen';
  });
});
```

Form submission (with the file attached) works with a plain
`<form enctype="multipart/form-data" method="post">` — no client-side
upload logic required.

## Props

| Prop | Type | Notes |
|---|---|---|
| `label` | `ReactNode` | required |
| `hint` / `error` / `required` | | |
| `accept` | `string` | native `accept` attribute, e.g. MIME types |
| `multiple` | `boolean` | allow more than one file |
| `chosenLabel` | `string` | default `'file chosen'` |
| `noFileLabel` | `string` | default `'No file chosen'` |
