# Select

A labelled native select for one choice from a **long** list — regions,
zones, categories.

<Example>
  <Select
    label="Region"
    placeholder="Choose a region"
    hint="Your region of residence"
    options={[
      { value: 'aa', label: 'Afar' },
      { value: 'am', label: 'Amhara' },
      { value: 'bg', label: 'Benishangul-Gumuz' },
      { value: 'or', label: 'Oromia' },
      { value: 'so', label: 'Somali' },
      { value: 'ti', label: 'Tigray' },
    ]}
  />
</Example>

```tsx
import { Select } from '@ethds/react';

<Select
  label="Region"
  placeholder="Choose a region"
  options={[{ value: 'or', label: 'Oromia' }, …]}
/>
```

## When to use this component

Use when there are too many options to show as [Radios](radios) (roughly
7+), such as the administrative lists in the
region → zone/sub-city → woreda address cascade.

## When not to use this component

Don't use for 2–6 options (radios let the citizen compare at a glance),
and don't rely on a placeholder as the label.

## Error state

<Example>
  <Select
    label="Region"
    placeholder="Choose a region"
    error="Select your region"
    options={[
      { value: 'aa', label: 'Afar' },
      { value: 'or', label: 'Oromia' },
    ]}
  />
</Example>

## Accessibility

- A **native `<select>`** — fully keyboard/screen-reader accessible and
  uses the platform picker on mobile (great on low-end devices).
- Visible associated label; hint/error via `aria-describedby`;
  `aria-invalid` on error; 44px minimum height.
- In dependent cascades, a child select is `disabled` until its parent is
  chosen — announced, not just greyed out.

## Plain HTML

Same `for`/`id`/`aria-describedby` wiring as
[Text input](text-input#plain-html); a disabled first `<option>` stands
in for a placeholder.

```html
<div class="ethds-field">
  <label for="region" class="ethds-label">Region</label>
  <span id="region-hint" class="ethds-hint">Your region of residence</span>
  <select id="region" name="region" class="ethds-select" aria-describedby="region-hint">
    <option value="" disabled selected>Choose a region</option>
    <option value="aa">Afar</option>
    <option value="am">Amhara</option>
    <option value="or">Oromia</option>
    <option value="ti">Tigray</option>
  </select>
</div>
```

```css
.ethds-select {
  min-height: 44px;
  padding: var(--ethds-space-2) var(--ethds-space-3);
  border: 1px solid var(--ethds-color-border);
  border-radius: var(--ethds-radius-md);
}
.ethds-select:focus-visible {
  outline: 2px solid var(--ethds-color-focus);
  border-color: var(--ethds-color-focus);
}
.ethds-select--error { border-color: var(--ethds-color-error); }
```

For a dependent cascade (region → zone → woreda), give the child
`<select>` the `disabled` attribute until its parent has a value, and
toggle it with a few lines of JavaScript on the parent's `change` event —
the `disabled` state itself is what gets announced to screen readers, no
extra ARIA needed.
