# Radios

Use radios when the citizen can select exactly **one** option from a
short list.

<Example>
  <RadioGroup
    legend="Where do you live?"
    options={[
      { value: 'aa', label: 'Addis Ababa' },
      { value: 'or', label: 'Oromia' },
      { value: 'am', label: 'Amhara' },
      { value: 'ti', label: 'Tigray' },
    ]}
  />
</Example>

```tsx
import { RadioGroup } from '@ethds/react';

<RadioGroup
  legend="Where do you live?"
  options={[
    { value: 'aa', label: 'Addis Ababa' },
    { value: 'or', label: 'Oromia' },
    { value: 'am', label: 'Amhara' },
    { value: 'ti', label: 'Tigray' },
  ]}
  onChange={(value) => …}
/>
```

## When to use this component

Use for a single choice from up to ~6 visible options — the citizen can
compare all options at once.

## When not to use this component

Don't use when multiple selections are allowed ([Checkbox](checkbox)) or
when the list is long ([Select](select)). Never preselect an answer to a
question the citizen must actually decide.

## Tile variant

Large bordered targets with optional descriptions (USWDS-style) — easier
to tap on small screens. Semantics are unchanged: still native radios in
a labelled group.

<Example>
  <RadioGroup
    legend="Select one historical figure"
    tile
    defaultValue="truth"
    options={[
      { value: 'truth', label: 'Sojourner Truth' },
      { value: 'douglass', label: 'Frederick Douglass', hint: 'This is optional text that can be used to describe the label in more detail.' },
      { value: 'washington', label: 'Booker T. Washington' },
    ]}
  />
</Example>

```tsx
<RadioGroup legend="Select one historical figure" tile options={…} />
```

## Accessibility

- A `<fieldset>` with a `<legend>` — the question is announced with the
  options as one group.
- Native radios: arrow keys move within the group, <kbd>Tab</kbd>
  enters/leaves — standard roving behaviour for free.
- Group hint/error via `aria-describedby`; error sets `aria-invalid` on
  the group; per-option hints are individually associated.

## Plain HTML

A `<fieldset>`/`<legend>` group of native radios sharing one `name` —
arrow-key navigation and toggling come from the browser for free.

```html
<fieldset class="ethds-radio-group">
  <legend class="ethds-legend">Where do you live?</legend>

  <div class="ethds-radio-options">
    <div class="ethds-radio-row">
      <input id="region-aa" name="region" type="radio" value="aa" class="ethds-radio" />
      <label for="region-aa" class="ethds-label">Addis Ababa</label>
    </div>
    <div class="ethds-radio-row">
      <input id="region-or" name="region" type="radio" value="or" class="ethds-radio" />
      <label for="region-or" class="ethds-label">Oromia</label>
    </div>
  </div>
</fieldset>

<!-- A per-option hint is linked the same way a field hint is: -->
<div class="ethds-radio-row">
  <input id="delivery-pickup" name="delivery" type="radio" value="pickup"
         class="ethds-radio" aria-describedby="delivery-pickup-hint" />
  <label for="delivery-pickup" class="ethds-label">Pick up</label>
</div>
<span id="delivery-pickup-hint" class="ethds-hint">Collect from your kebele office</span>
```

```css
.ethds-radio-group { margin: 0; padding: 0; border: 0; }
.ethds-legend { font: var(--ethds-font-medium) var(--ethds-text-base) var(--ethds-font-sans); margin-bottom: var(--ethds-space-2); }
.ethds-radio-options { display: flex; flex-direction: column; gap: var(--ethds-space-2); }
.ethds-radio-row { display: flex; align-items: flex-start; gap: var(--ethds-space-2); }
.ethds-radio { width: 20px; height: 20px; margin: 0; accent-color: var(--ethds-color-primary); }
.ethds-radio:focus-visible { outline: 2px solid var(--ethds-color-focus); outline-offset: 2px; }
```

On error, set `aria-describedby`/`aria-invalid="true"` on the
`<fieldset>` (not the individual inputs) and add an error span after the
options — same group-level convention shown in the "Hints, per-option
hints, and error" example above. The tile variant is a CSS-only
extension (a bordered box that highlights `:has(.ethds-radio:checked)`)
and isn't required for a functioning group.

## Props

| Prop | Type | Notes |
|---|---|---|
| `legend` | `ReactNode` | required — the group question |
| `options` | `{ value, label, hint?, disabled? }[]` | required |
| `value` / `defaultValue` | `string` | controlled / uncontrolled |
| `onChange` | `(value: string) => void` | |
| `hint` / `error` / `required` | | group-level |
| `tile` | `boolean` | USWDS-style tiles, default `false` |
