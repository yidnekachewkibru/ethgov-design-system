# Notification

A transient, dismissible message announced when it appears — "saved",
"action needed", connection notices.

<Example>
  <NotificationDemo />
</Example>

```tsx
import { Notification } from '@ethds/react';

<Notification
  variant="success"
  iconLabel="Success"
  title="Saved"
  dismissLabel="Dismiss"
  onDismiss={() => …}
>
  Your changes have been saved.
</Notification>
```

## When to use this component

Use for messages that arrive **while the citizen is doing something
else** — an async result, a dashboard "action needed" banner — and that
they may dismiss.

## When not to use this component

Don't use for messages that are part of the page's core content
([Alert](alert)), and never auto-dismiss a message the citizen must act
on.

## Accessibility

- Wrapped in a live region: `role="alert"` for `error`, `role="status"`
  otherwise — announced without moving focus (WCAG 4.1.3).
- The dismiss button has an accessible name (`dismissLabel`, translated);
  severity is carried by text + labelled icon, with the coloured edge as
  reinforcement only.

## Plain HTML

```html
<div role="status" class="ethds-notification ethds-notification--success">
  <span class="ethds-notification__icon" role="img" aria-label="Success">✓</span>
  <div class="ethds-notification__body">
    <p class="ethds-notification__title">Saved</p>
    <div>Your changes have been saved.</div>
  </div>
  <button type="button" class="ethds-notification__dismiss" aria-label="Dismiss">
    <span aria-hidden="true">×</span>
  </button>
</div>
```

Use `role="alert"` instead of `role="status"` only for the `error`
variant — everything else about the markup is identical across variants,
only the modifier class (`--info` / `--success` / `--warning` /
`--error`) changes which border colour is applied.

```css
.ethds-notification {
  display: flex;
  gap: var(--ethds-space-3);
  padding: var(--ethds-space-3) var(--ethds-space-4);
  border: 1px solid transparent;
  border-radius: var(--ethds-radius-md);
  box-shadow: var(--ethds-elevation-2);
  background: var(--ethds-white);
}
.ethds-notification--info { border-left: 4px solid var(--ethds-color-info); }
.ethds-notification--success { border-left: 4px solid var(--ethds-color-success); }
.ethds-notification--warning { border-left: 4px solid var(--ethds-color-warning); }
.ethds-notification--error { border-left: 4px solid var(--ethds-color-error); }
.ethds-notification__title { margin: 0; font-weight: var(--ethds-font-bold); }
.ethds-notification__dismiss {
  width: 32px; height: 32px; padding: 0; border: 0; background: transparent;
  color: var(--ethds-color-text-secondary); cursor: pointer;
}
.ethds-notification__dismiss:focus-visible {
  outline: 2px solid var(--ethds-color-focus); outline-offset: 2px;
}
```

To actually dismiss the notification without JavaScript, render it
server-side only when there's something to announce; with a sprinkle of
JS, wire the dismiss button to remove its own `.ethds-notification` node
from the DOM (`event.currentTarget.closest('.ethds-notification').remove()`).

## Props

| Prop | Type | Default |
|---|---|---|
| `variant` | `'info' \| 'success' \| 'warning' \| 'error'` | `'info'` |
| `title` | `ReactNode` | optional |
| `iconLabel` | `string` | accessible icon name |
| `dismissLabel` | `string` | shows the dismiss button when set |
| `onDismiss` | `() => void` | |
