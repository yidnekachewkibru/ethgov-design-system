# Textarea

A labelled multi-line field for longer answers — descriptions,
complaints, additional details.

<Example>
  <TextArea label="Describe what happened" hint="Up to 500 characters" rows={4} />
</Example>

```tsx
import { TextArea } from '@ethds/react';

<TextArea label="Describe what happened" hint="Up to 500 characters" rows={4} />
```

## When to use this component

Use when the citizen needs to write more than one line, in their own
words and their own language — the field supports full Unicode including
Ge'ez script.

## When not to use this component

Don't use for short single-line answers ([Text input](text-input)) or
for structured data that should be separate fields.

## Error state

<Example>
  <TextArea label="Details" error="This field is required" rows={3} required />
</Example>

## Accessibility

- Same wiring as [Text input](text-input): associated visible label,
  hint/error via `aria-describedby`, `aria-invalid` on error.
- Vertically resizable so citizens can see more of their text.
- Generous line-height for readability, including Ge'ez script.

## Plain HTML

Same `for`/`id` and `aria-describedby` wiring as
[Text input](text-input#plain-html) — only the field element changes.

```html
<div class="ethds-field">
  <label for="details" class="ethds-label">Describe what happened</label>
  <span id="details-hint" class="ethds-hint">Up to 500 characters</span>
  <textarea
    id="details"
    name="details"
    rows="4"
    class="ethds-textarea"
    aria-describedby="details-hint"
  ></textarea>
</div>
```

```css
.ethds-textarea {
  width: 100%;
  padding: var(--ethds-space-2) var(--ethds-space-3);
  font: var(--ethds-text-base) var(--ethds-font-sans);
  line-height: var(--ethds-leading-relaxed);
  border: 1px solid var(--ethds-color-border);
  border-radius: var(--ethds-radius-md);
  resize: vertical;
}
.ethds-textarea:focus-visible {
  outline: 2px solid var(--ethds-color-focus);
  border-color: var(--ethds-color-focus);
}
.ethds-textarea--error { border-color: var(--ethds-color-error); }
```

On error, add `class="ethds-textarea--error"`, `aria-invalid="true"`, and
an `id="details-error"` span (added to `aria-describedby`) exactly as
shown for Text input.
