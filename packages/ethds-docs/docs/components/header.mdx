# Header

The government identity and primary navigation — the consistent banner
that tells a citizen this is an official service.

<Example>
  <Header
    serviceName="Federal Revenue Service"
    homeHref="#"
    homeLabel="Federal Revenue Service — home"
    nav={[
      { label: 'Home', href: '#', current: true },
      { label: 'Services', href: '#' },
      { label: 'News', href: '#' },
      { label: 'Contact', href: '#' },
    ]}
    actions={<LanguageSwitcherDemo />}
  />
</Example>

```tsx
import { Header, LanguageSwitcher } from '@ethds/react';

<Header
  serviceName="Federal Revenue Service"

  nav={[{ label: 'Home', href: '/', current: true }, …]}
  actions={<LanguageSwitcher … />}
/>
```

## When to use this component

On **every page** of every ETHDS service. The shared identity — Star Blue
banner, emblem slot, consistent navigation — is how citizens recognise
official services and spot impersonation.

## When not to use this component

Don't rebuild a per-ministry header with its own colours or structure —
the delivering body is named *within* the shared identity, per the
government branding standards.

## Accessibility

- A `<header>` banner landmark; the identity is a home link with an
  explicit accessible name (`homeLabel`).
- The primary nav is a labelled `<nav>`; the current item has
  `aria-current="page"`.
- White-on-Star-Blue text is 8.31:1; focus rings are visible against the
  blue.
- Put the [Language switcher](language-switcher) in `actions` so it's on
  every page, early in the tab order.

## Plain HTML

This is the same header markup already used in the
[404 page](/docs/templates/404-page#plain-html) template — every ETHDS
page shares this one structure rather than a per-page rebuild.

```html
<header class="ethds-header">
  <div class="ethds-header__bar">
    <a href="/" class="ethds-header__identity" aria-label="Federal Revenue Service — home">
      <span class="ethds-header__service-name">Federal Revenue Service</span>
    </a>
    <div class="ethds-header__actions">
      <!-- Language switcher goes here — see its own Plain HTML section. -->
    </div>
  </div>
  <nav aria-label="Primary" class="ethds-header__nav">
    <ul class="ethds-header__nav-list">
      <li><a href="/" class="ethds-header__nav-link ethds-header__nav-link--current" aria-current="page">Home</a></li>
      <li><a href="/services" class="ethds-header__nav-link">Services</a></li>
      <li><a href="/news" class="ethds-header__nav-link">News</a></li>
      <li><a href="/contact" class="ethds-header__nav-link">Contact</a></li>
    </ul>
  </nav>
</header>
```

`aria-current="page"` and the modifier class server-side, based on the
route being rendered.

## Props

| Prop | Type | Notes |
|---|---|---|
| `serviceName` | `ReactNode` | required, translatable |
| `homeHref` / `homeLabel` | `string` | identity link + its accessible name |
| `logo` | `ReactNode` | emblem slot |
| `nav` | `{ label, href, current? }[]` | primary navigation |
| `navLabel` | `string` | nav landmark name |
| `actions` | `ReactNode` | e.g. LanguageSwitcher, account |
