## When to use this component

On every page. Standard homes for the accessibility statement, privacy,
contact, and cross-government links.

## When not to use this component

Don't bury primary tasks in the footer — it's for secondary links.

## Accessibility

- A `<footer>` contentinfo landmark; each link group is its own labelled
  `<nav>` with a heading, so screen-reader users can navigate the footer
  by section.
- Link colours on the dark surface meet AA; focus rings are visible.

## Plain HTML

The [404 page](/docs/templates/404-page#plain-html) template's footer is
the minimal case (just a copyright line); this is the full version with
grouped link navs.

```html
<footer class="ethds-footer" aria-label="Footer">
  <div class="ethds-footer__groups">
    <nav aria-label="Services" class="ethds-footer__group">
      <h2 class="ethds-footer__title">Services</h2>
      <ul class="ethds-footer__list">
        <li><a href="/services" class="ethds-footer__link">All services</a></li>
        <li><a href="/track" class="ethds-footer__link">Track an application</a></li>
      </ul>
    </nav>
    <nav aria-label="About" class="ethds-footer__group">
      <h2 class="ethds-footer__title">About</h2>
      <ul class="ethds-footer__list">
        <li><a href="/contact" class="ethds-footer__link">Contact us</a></li>
        <li><a href="/accessibility" class="ethds-footer__link">Accessibility statement</a></li>
        <li><a href="/privacy" class="ethds-footer__link">Privacy</a></li>
      </ul>
    </nav>
  </div>
  <p class="ethds-footer__copyright">© 2026 Government of Ethiopia. Built with ETHDS.</p>
</footer>
```

Static markup, no JavaScript. Each group's `aria-label` matches its
visible `<h2>` so a screen reader announces the section by name when
jumping between the footer's landmarks.

## Props

| Prop | Type | Notes |
|---|---|---|
| `groups` | `{ title, links: { label, href }[] }[]` | grouped link navs |
| `copyright` | `ReactNode` | legal line |
| `label` | `string` | optional landmark name |
