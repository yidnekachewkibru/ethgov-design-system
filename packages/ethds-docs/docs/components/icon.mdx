# Icon

An accessible SVG icon on a 24×24 grid. Icons support comprehension —
they never carry meaning alone.

<Example>
  <div style={{ display: 'flex', gap: '1.5rem', color: 'var(--ethds-color-primary)' }}>
    <Icon label="Check">{CheckIcon}</Icon>
    <Icon label="Information">{InfoIcon}</Icon>
    <Icon label="Warning">{WarningIcon}</Icon>

    <Icon label="Search">{SearchIcon}</Icon>

  </div>
</Example>

```tsx
import { Icon, CheckIcon } from '@ethds/react';

// Decorative (hidden from screen readers — a text label carries meaning):
<span>Done <Icon>{CheckIcon}</Icon></span>

// Meaningful (announced with an accessible name, in the active language):
<Icon label="Success">{CheckIcon}</Icon>
```

## When to use this component

Use icons **next to text** to aid recognition (status, actions,
categories) — especially helpful across languages and literacy levels.

## When not to use this component

Don't use an icon as the only signal of state or the only label of a
control. Icon-only buttons must have an `aria-label`.

## Decorative vs meaningful

- **No `label`** → decorative: `aria-hidden="true"`, not announced.
- **With `label`** → meaningful: `role="img"` + `aria-label`, announced.
  Pass the label in the citizen's active language.

## Accessibility

- Inherits `currentColor`, so icons automatically meet the same contrast
  as the text beside them.
- A standalone meaningful icon must meet 3:1 non-text contrast.
- `focusable={false}` so decorative SVGs never catch keyboard focus.

## Plain HTML

```html
<!-- Decorative — hidden from screen readers, text carries the meaning: -->
<span>Done
  <svg class="ethds-icon" viewBox="0 0 24 24" fill="none" stroke="currentColor"
       stroke-width="2" stroke-linecap="round" stroke-linejoin="round"
       aria-hidden="true" focusable="false">
    <path d="M20 6 9 17l-5-5" />
  </svg>
</span>

<!-- Meaningful — announced with an accessible name: -->
<svg class="ethds-icon" viewBox="0 0 24 24" fill="none" stroke="currentColor"
     stroke-width="2" stroke-linecap="round" stroke-linejoin="round"
     role="img" aria-label="Success" focusable="false">
  <path d="M20 6 9 17l-5-5" />
</svg>
```

```css
.ethds-icon { display: inline-block; vertical-align: middle; width: var(--ethds-icon-md); height: var(--ethds-icon-md); color: inherit; }
.ethds-icon--sm { width: var(--ethds-icon-sm); height: var(--ethds-icon-sm); }
.ethds-icon--lg { width: var(--ethds-icon-lg); height: var(--ethds-icon-lg); }
.ethds-icon--xl { width: var(--ethds-icon-xl); height: var(--ethds-icon-xl); }
```

`stroke="currentColor"` is what makes an icon inherit the surrounding
text's colour (and therefore its contrast) automatically. The path data
above is the actual Check icon shipped with `@ethds/react` — copy the
`<path>` from any icon you need out of
[`Icon/icons.tsx`](https://github.com/yidnekachewkibru/ethgov-design-system/blob/main/packages/ethds-react/src/components/Icon/icons.tsx).

## Props

| Prop | Type | Default |
|---|---|---|
| `children` | SVG content (24×24, stroke-based) | required |
| `label` | `string` — accessible name; omit for decorative | — |
| `size` | `'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'` |
