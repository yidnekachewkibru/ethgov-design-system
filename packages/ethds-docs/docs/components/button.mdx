# Button

Use a button to help citizens carry out an action, like starting an
application, saving progress, or submitting a form.

<Example>
  <Button variant="primary">Start now</Button>
</Example>

```tsx
import { Button } from '@ethds/react';

<Button variant="primary">Start now</Button>
```

## When to use this component

Use a button for **actions** — starting a service, submitting, saving,
cancelling. Write the label as a verb describing what happens, in the
citizen's language ("Start now", "Submit complaint", "Pay ETB 350.00").

## When not to use this component

Don't use a button for **navigation** to another page — use a
[Link](link). Don't disable buttons to communicate "not yet valid";
prefer letting the citizen submit and showing clear errors.

## Variants

<Example>
  <div style={{ display: 'flex', gap: '1rem', flexWrap: 'wrap' }}>
    <Button variant="primary">Primary</Button>
    <Button variant="secondary">Secondary</Button>
    <Button variant="ghost">Ghost</Button>
    <Button variant="danger">Delete</Button>
    <Button disabled>Disabled</Button>
  </div>
</Example>

- **primary** — the main call to action; one per screen.
- **secondary** — alternative actions (Back, Cancel).
- **ghost** — low-emphasis actions in dense UI.
- **danger** — destructive actions; pair with a confirmation step.

## Accessibility

- Renders a native `<button>` — keyboard operable (<kbd>Enter</kbd>/<kbd>Space</kbd>)
  with correct semantics for free.
- Defaults to `type="button"` so it never submits a form by accident.
- Visible focus ring (`--ethds-color-focus`); minimum 44px height —
  comfortably above the 24px WCAG 2.2 target minimum.
- The accessible name is the button text — pass translated text as
  children. Icon-only buttons must supply an `aria-label`.

## Plain HTML

Not on React? A button is a native `<button>` with `type="button"` (so it
never submits a form by accident) styled with
[`@ethds/tokens`](https://github.com/yidnekachewkibru/ethgov-design-system/tree/main/packages/ethds-tokens)
variables — no JavaScript required. This is a starter you adapt and
maintain yourself, not a package ETHDS ships and versions for you.

```html
<link rel="stylesheet" href="path/to/@ethds/tokens/dist/tokens.css" />

<button type="button" class="ethds-button ethds-button--primary">
  Start now
</button>
```

```css
.ethds-button {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: var(--ethds-space-2);
  min-height: 44px; /* comfortably above the 24px WCAG 2.2 target minimum */
  padding: var(--ethds-space-3) var(--ethds-space-4);
  border: 1px solid transparent;
  border-radius: var(--ethds-radius-md);
  font: var(--ethds-font-medium) var(--ethds-text-base) var(--ethds-font-sans);
  cursor: pointer;
}
.ethds-button:focus-visible {
  outline: 2px solid var(--ethds-color-focus);
  outline-offset: 2px;
}
.ethds-button:disabled {
  cursor: not-allowed;
  opacity: 0.5;
}

.ethds-button--primary {
  background: var(--ethds-color-primary);
  color: var(--ethds-white);
}
.ethds-button--secondary {
  background: var(--ethds-white);
  color: var(--ethds-color-primary);
  border-color: var(--ethds-color-primary);
}
.ethds-button--ghost {
  background: transparent;
  color: var(--ethds-color-interactive);
}
.ethds-button--danger {
  background: var(--ethds-color-error);
  color: var(--ethds-white);
}
```

The accessible name is whatever text is inside the tag — keep it
translated per locale, the same as the React version's `children`. For an
icon-only button, add `aria-label="…"` since there's no visible text to
name it.

## Props

| Prop | Type | Default |
|---|---|---|
| `variant` | `'primary' \| 'secondary' \| 'ghost' \| 'danger'` | `'primary'` |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` |
| `fullWidth` | `boolean` | `false` |
| …plus all native `<button>` props | | |
