# Language switcher

How a citizen chooses their language. Every language is listed in **its
own name and script** (autonym), so a speaker finds theirs without
reading the current language.

<Example>
  <LanguageSwitcherDemo />
</Example>

```tsx
import { LanguageSwitcher } from '@ethds/react';

<LanguageSwitcher
  label="Choose language"
  value={locale}
  onChange={setLocale}
  languages={[
    { code: 'en', label: 'English' },
    { code: 'am', label: 'አማርኛ' },
    { code: 'om', label: 'Afaan Oromoo' },
    { code: 'ti', label: 'ትግርኛ' },
    { code: 'so', label: 'Soomaali' },
    { code: 'aa', label: 'Qafar' },
  ]}
/>
```

## When to use this component

On **every page**, in the [Header](header)'s `actions` slot — a citizen
who can't read the current language must be able to find it immediately.

## When not to use this component

Don't represent languages with flags (flags are countries, not
languages) and don't hide the switcher behind an unlabelled icon.

## Behaviour rules

Per the localization framework's switcher standard:

1. Switching applies **immediately** to content *and* formatting, keeping
   the citizen on the same page with their input intact.
2. The choice **persists** across the journey and visits.
3. The document `lang` updates so screen readers switch pronunciation.

## Accessibility

- A labelled native `<select>` — keyboard and screen-reader accessible,
  platform picker on mobile.
- Each `<option>` sets `lang` to its own locale so its autonym is
  pronounced correctly (e.g. አማርኛ announced in Amharic).

## Plain HTML

Belongs in the [Header](header#plain-html)'s `.ethds-header__actions`
slot, on every page.

```html
<div class="ethds-language-switcher">
  <label for="lang" class="ethds-language-switcher__label">Choose language</label>
  <select id="lang" name="lang" class="ethds-language-switcher__select" onchange="location.href = this.value">
    <option value="/en" lang="en" selected>English</option>
    <option value="/am" lang="am">አማርኛ</option>
    <option value="/om" lang="om">Afaan Oromoo</option>
    <option value="/ti" lang="ti">ትግርኛ</option>
    <option value="/so" lang="so">Soomaali</option>
    <option value="/aa" lang="aa">Qafar</option>
  </select>
</div>
```

```css
.ethds-language-switcher { display: inline-flex; align-items: center; gap: var(--ethds-space-2); }
.ethds-language-switcher__label { font-size: var(--ethds-text-sm); color: inherit; }
.ethds-language-switcher__select {
  min-height: 40px;
  padding: var(--ethds-space-1) var(--ethds-space-3);
  border: 1px solid var(--ethds-color-border);
  border-radius: var(--ethds-radius-md);
}
.ethds-language-switcher__select:focus-visible {
  outline: 2px solid var(--ethds-color-focus);
  outline-offset: 2px;
}
```

Without a client framework, each `<option>`'s `value` is the URL for
that language's version of the current page, and a one-line
`onchange="location.href = this.value"` navigates there — the server
renders the page in that locale and sets `<html lang>` accordingly,
satisfying the same three behaviour rules above without any client-side
state.

## Props

| Prop | Type | Notes |
|---|---|---|
| `label` | `string` | required — accessible name, in the active language |
| `languages` | `{ code, label }[]` | autonym labels |
| `value` / `onChange` | `string` / `(code) => void` | controlled |
